### Key Insights and Progress Log.  
>In this Chapter we started with learning how to loop through an entire list, looping allows us to take the same action, or set of actions with every item in the list.
>When you want to do the same action with every item in a list, you can use Python's `for` loop, say if your code has the line `for <variable_name> in <list_name>:` pressing enter will automatically indent the next lines of code you write till you delete the preceding whitespace. Python reads the first line of the loop  "`for <variable_name> in <list_name>:`", this tells python to retrieve the first value from the list <list_name> and associate it with the variable <variable_name>, the in the indentatied lines below the first line, we can use the variable like want, if the list containms more values, python jumps back to the first line "`for <variable_name> in <list_name:`" and retrieves the next item in the specified list and associates it to the specified variable, runs the below correctly indented likes of code and repeat and repeat till the last item of the list, runs the indented code for that and drops out of the loop, the specified <variable_name> is still associated with the last value of the specified list!. (each indended line below the "for, in" statement is considered inside a loop).
>Python uses indentation to determine how a line, or a group of lines is related to the rest of the program, the use of indentation also makes the code very easy to read. An `Indentation error` might arise when we indent likes that donesnt need to be indented or forget to indent items that need to be indented, example, forgetting to indent after a for statement in a loop which returns an Indentation Wrror in the terminal, Forgetting to indent additional lines in a for statement might make the code do smth that we dont want it to do, like doing an operation with only the last value of the list (ponder), no returned error tho if atleast one line after the for statement is indented, just not expected results, this is called a `logic error`, the synax is valif python code but we didnt produce desirable results, next is indenting unnecessarily, which will also be reported as an `IndentationError` by python, tracebacked as `unexpected indent`, indenting unnecessarily after the loop may run a line for all items in the list, i.e... the action will be repeated for all items in the list, the specified variable in the for statement labelling the respecive all values, no matter if you used that vraiable in the line that you unnecessarily indented, sometimes it might be reported as an error and sometimes will remain just a plain old unreported logical error if the syntax is valid, finally comes the error of `forgetting the colon` in the for statement, first it wont automatically indent after you press enter and if you try to do it yourself, it will still reurn a `SyntaxError` as expected ':'.
