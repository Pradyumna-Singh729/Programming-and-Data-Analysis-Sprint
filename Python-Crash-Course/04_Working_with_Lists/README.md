>Lists are also ideal for storing sets of numbers, and Python provides a variety of tools to help us work efficiently with lists of numbers.
>Python's `range()` function makes it easy to generate a series of numbers. For example, `for value in range(n,m):` (where n,m are integers and n<m) will start a for loop and associate values n, n+1, n+2,...m-1.  to the variable `value` in the loop and run the indented code for all of them, except the second value we provided in the argument of `range()`, since the range() function causes python to start counting at the first value we give it and it stops when it reaches the second value provided, so it stops the loop before implementing the indented code of the for statement to the second value provided.  We can also feed only one integer in the argument of range() and it will start the sequence of numbers from 0, example range(6) will return all the numbers from 0 through 5.
>If we want to make a list of numbers, we can convert the results of `range()` directly into a list using the `list()` Function. When we wrap `list()` around a call to the range() function, the output will be a list of numbers. For example, `<list_name> = list(range(n,m))` will assign the variable <list_name> to a list of numbers from n through m-1. Also if we feed the `range()` function with a third argument, the it returns numbers starting at the first argument, increasing with a step size of the third argument, through all the numbers less than the second argument, for example `range(2, 11, 2)` (or even `range(2,10,2)`) will return the numbers 2, 4, 6, 8, 10.  We can create almost any set of numbers we want using the `range()` function (ponder making a list of perfect squares using a for loop and the append method after creating an empty list, know that in the append method's argument, you can feed a plain variable, a list or even an operation applied on a variable like `squares.append(value**2)` etc).  We can also skip the for loop entirely to generate a list of numbers and use the `list comprehensions`. A list comprehension allows us to generate the same list we did with the for loop with just one line of code, it combines the `for` loop and the creation of new elements into one line, and automatically appends each new element, for example `squares = [value**2 for value in range(1,11)]`  will create a list names squares AND fill it with perfect squares from 1 through 100.  The simple way to use this syntax is to begin with a name for the list, then open a set of square brackets and define the expression for the values we want to store in the new list, the write a `for` loop to generate the numbers we want to feed in the expression and close the square brackets.
>Next we can also use simple python functions to do simple statistics with a list of numbers, we have `min(<list_name>)', `max(<list_name>)`, `sum(<list_name>)` to return minimum/maximum or the sum of all numbers in a list of numbers named <list_name>, example [1, 2 , 3, 4, 5, 6] etc.

>We can also work with a specific group of items, i.e... a subset of a lost, called a `slice` in python, instead of working with a single element or an entire list.
>To make a slice, we specify the index of the first and the last elements we want to work with, as with the range() function, python stops one item before the second index we specify, so if we use `<list_name>(n:m)`, it will return the items in the list indexed at n, through m-1, example `<list_name>(0:3)` will return the first 3 items in the specified list, i.e... valued indexed at 0, 1 and 2 so the first, second and third element. If we omit the first index in a slice, Python automatically starts the slice at the beginning of the slice, i.e... from the item indexed 0, e.g... `<list_name>(:n)` will return the first n items of the list. A similar syntax works if we want a slice that includes the end of the list, just omit the second index, e.g... `<list_name>(n:)` will return all the items indexed n, all the way to where the list ends, `<list_name>(4:)` will return all the items from the item indexed 4 to the last element of the list, `<list_name>(-7:)` will unsurprisingly return the last 7 items of the list, items indexed -7 through -1. Finally if a third index is included, the python sets it up as a step size starting at the first index.
>We can easily loop through a slice instead of the entire list with the `for` statement like `for item in items[:3]` which loops through only the first 3 elements of the list "items".
>Another application of the slize index is `copying a list`. To copy a list, we can make a slice the includes the entire original list by omitting the first index and the second index ([:]). This tells python to make a slice the starts at the first item and end with the last item, producing a copy of the entire list, for example `<list_name_original>` is a existing list and we want to make a seperate list which is the copy of this list, then we can do so by simple `<list_name_duplicate> = <list_name_original>[:]`, know that we have now 2 seperate independent lists named <list_name_duplicate> and <list_name_original> which have the exact same elements (could have make an independant list with a subset of the items in the original list too). Notice that if we simply set <list_name_duplicate> equal to <list_name_original> without the slice syntax, i.e... doing `<list_name_duplicate> = <list_name_original>` then we would not produce 2 seperate lists, but the same list (the original one) will now have 2 sepearate lables, the significance is that now if we use any of the variable name to do an operation, it will happen to the same list. Got 1 list with 2 names rather than 2 lists that have the same elements and different names. (got 2 var pointing to the same object).

>Sometimes we want to create a list of items that cannot change. Lists are Dynamic, but `Tuples` allow us to do just that. Python refers to values that can't change as `immutable`, and an immputable list is called a `tuple`.
>A tuple looks just like a list, except we use parenthesis instead of square brackets. Once we define a tuple, we can access the individual elements using each item's index, just as we would for a list, e.g... `<tuple_name> = (1, 2, 3, 4)` defines a tuple and we can access the... say third element by `<tuple_name>[2]`, now if we try to modiify the tuple by using a line like `<tuple_name>[3] = 5` then python will return a `type error` Because we are trying to alter a tuple, which cant be done to that type of object.  Also Tuples are technically defined by the presence of a comma, the parenthesis just make them look neater and more readable. If we want to define a tuple with one element, we need to include the trailing comma, e.g... `my_t = (3,)`.
>We can also loop over all the values in a tuple using a for loop, just as we did with a list, i.e... simply using the for statement `for <variable_name> in <tuple_name>:`

>Styling Your Code (PEP 8 Guidelines)
Code is read much more often than it is written. Writing clean, consistent code helps you keep track of what your programs are doing and makes it easier to debug and collaborate.

The Style Guide (PEP 8): Python Enhancement Proposals (PEP) are documents detailing changes or standards for the language. PEP 8 is the definitive style guide for writing clean Python.

Indentation: Use exactly 4 spaces per indentation level. Never mix tabs and spaces in a file—the Python interpreter gets confused, leading to errors that are very difficult to diagnose. Set your text editor to automatically convert the TAB key into spaces.

Line Length: Keep code lines under 80 characters (specifically a maximum of 79 characters). This historical standard allows you to keep multiple files open side-by-side on a screen without hiding text. Keep comments limited to 72 characters per line, as documentation tools often add characters at the start of comment lines. Use a vertical line visual cue in your editor to track this.

Blank Lines: Use blank lines to group distinct sections of your program visually, but do not use them excessively (avoid placing 3 or 4 blank lines between code blocks). Vertical spacing is completely disregarded by the Python interpreter and will not affect how your code runs, but it directly impacts readability.
